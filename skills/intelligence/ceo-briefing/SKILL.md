# CEO Briefing - Professional Skill

Autonomously generates executive-level business briefings by analyzing vault data, completed tasks, and key metrics.

## When to Use

- Scheduled: Every Monday morning at 7:00 AM
- User command: `/ceo-briefing`, "generate briefing", "weekly summary"
- Orchestrator-triggered: Periodic business review
- End of period: Monthly, quarterly summaries

## Before Implementation

| Source | Gather |
|--------|--------|
| **Dashboard.md** | Current status, pending items |
| **Done/** | Completed tasks this period |
| **Business_Goals.md** | Targets, KPIs, objectives |
| **Logs/** | Action logs, metrics |
| **Accounting/** | Financial data (if available) |

## Workflow

### Phase 1: Data Collection

```
1. Read Business_Goals.md for targets and KPIs
2. Scan Done/ folder for completed items
3. Read Dashboard.md for current status
4. Parse Logs/ for metrics and actions
5. Check Accounting/ for financial data
6. Scan Pending_Approval/ for blocked items
```

### Phase 2: Analysis

```yaml
Analysis Categories:
  revenue:
    - Calculate total revenue (if data available)
    - Compare to targets
    - Identify trends

  tasks:
    - Count completed tasks
    - Average completion time
    - Task breakdown by category

  productivity:
    - Response times
    - Processing volume
    - Bottlenecks identified

  issues:
    - Blocked items
    - Pending approvals
    - Escalations needed

  suggestions:
    - Cost optimization opportunities
    - Process improvements
    - Risk alerts
```

### Phase 3: Report Generation

```
1. Calculate metrics
2. Identify highlights and concerns
3. Generate proactive suggestions
4. Create formatted briefing document
5. Save to Briefings/ folder
6. Update Dashboard with summary
```

```python
def calculate_revenue_metrics(accounting_data, goals):
    current_revenue = sum(t['amount'] for t in accounting_data['transactions'])

    return {
        'total': current_revenue,
        'mtd_percentage': (current_revenue / goals['monthly_target']) * 100,
        'trend': calculate_trend(accounting_data, weeks=4),
        'projection': project_monthly(current_revenue, days_remaining)
    }
```

### Task Performance

```python
def calculate_task_metrics(done_folder, period):
    tasks = parse_completed_tasks(done_folder, period)

    return {
        'total_completed': len(tasks),
        'by_category': group_by_category(tasks),
        'avg_completion_time': calculate_avg_time(tasks),
        'completion_rate': len(tasks) / (len(tasks) + count_pending())
    }
```

### Response Time

```python
def calculate_response_times(logs):
    response_times = []

    for action in logs:
        if action['type'] == 'response':
            response_time = action['responded_at'] - action['received_at']
            response_times.append(response_time)

    return {
        'average': mean(response_times),
        'median': median(response_times),
        'within_target': sum(1 for r in response_times if r < TARGET) / len(response_times)
    }
```

### Cost Optimization

```python
def generate_cost_suggestions(subscriptions, usage_data):
    suggestions = []

    for sub in subscriptions:
        # Check if subscription is unused
        if sub['last_used'] > 30_days_ago:
            suggestions.append({
                'type': 'cost_saving',
                'item': sub['name'],
                'savings': sub['monthly_cost'],
                'reason': f"No usage in {days_since_use} days",
                'action': "Cancel subscription?"

        # Check for duplicate functionality

            suggestions.append({

                'item': sub['name'],
```

### Bottleneck Detection

```python
def detect_bottlenecks(task_data, logs):
    bottlenecks = []

    # Check for tasks taking too long
    for task in task_data:
        if task['completion_time'] > EXPECTED_TIME:
            bottlenecks.append({
                'type': 'slow_task',
                'item': task['name'],
                'impact': f"Took {task['completion_time']} vs expected {EXPECTED_TIME}",
                'suggestion': "Review process for optimization"
            })

    # Check for pending approvals
    pending = count_pending_approvals()
    if pending > THRESHOLD:
        bottlenecks.append({
            'type': 'approval_backlog',
            'item': f"{pending} items pending approval",
            'impact': "Delays in execution",
            'suggestion': "Review and approve pending items"
        })

    return bottlenecks
```

### Risk Alerts

```python
def check_risk_alerts(data):
    alerts = []

    # Check for overdue payments
    if data.get('overdue_payments'):
        alerts.append({
            'type': 'financial',
            'severity': 'high',
            'message': f"{len(data['overdue_payments'])} overdue payments totaling ${sum(data['overdue_payments'])}"
        })

    # Check for unresponded urgent items
    if data.get('urgent_unresponded'):
        alerts.append({
            'type': 'communication',
            'severity': 'high',
            'message': f"{len(data['urgent_unresponded'])} urgent items awaiting response"
        })

    # Check for subscription cost increases
    if data.get('subscription_cost_increase'):
        alerts.append({
            'type': 'cost',
            'severity': 'medium',
            'message': f"Subscription costs increased by {data['subscription_cost_increase']}%"
        })

    return alerts
```
