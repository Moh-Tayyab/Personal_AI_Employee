# Request Approval - Agent Skill

Creates structured approval requests for actions that exceed autonomy thresholds, enabling human-in-the-loop control over sensitive operations.

## When to Use

- Actions exceeding autonomy Level 2 thresholds
- Before executing sensitive actions (payments, legal, new contacts)
- When action risk score exceeds configured limits
- User commands: `/request-approval`, "need approval for"

## Before Implementation

| Source | Gather |
|--------|--------|
| **Company_Handbook.md** | Approval thresholds, autonomy matrix |
| **Business_Goals.md** | Financial limits, authorized contacts |
| **Pending_Approval/** | Check for existing similar requests |
| **Approved/ & Rejected/** | Check for precedent decisions |

## Workflow

### Phase 1: Analysis
1. Parse action type and parameters
2. Check against autonomy matrix
3. Calculate risk score
4. Determine if approval required

### Phase 2: Risk Assessment
Score each risk factor:

| Risk Factor | Points |
|-------------|--------|
| New contact (not in known contacts) | +2 |
| Financial commitment > $50 | +3 |
| Legal implications | +5 |
| External communication (new) | +2 |
| Data sharing | +3 |
| Reputation risk | +2 |

**Total Score:**
- 0-2: Level 2 (notify only)
- 3-5: Level 3 (approval required)
- 6+: Level 3 + urgent review

### Phase 3: Request Creation
Create approval request file:
```yaml
---
type: approval_request
action: <action_type>
created: <timestamp>
expires: <24h from now>
priority: <priority>
status: pending
risk_score: <calculated_score>
---
```

### Phase 4: Notification
1. Write to `Pending_Approval/`
2. Update Dashboard.md with pending count
3. Log request creation

## Approval Rules Matrix

| Action | Auto-Approve | Requires Approval | Notes |
|--------|--------------|-------------------|-------|
| Email to known contact | ✅ | ❌ | Level 2, notify |
| Email to new contact | ❌ | ✅ | Level 3 |
| Social posts (scheduled) | ✅ | ❌ | Level 2, notify |
| Social replies/DMs | ❌ | ✅ | Level 3 |
| Payments < $50 | ❌ | ✅ | Level 3 |
| Payments > $50 | ❌ | ✅ Always | Level 3 + urgent |
| Calendar events (known) | ✅ | ❌ | Level 1 |
| Calendar events (new) | ❌ | ✅ | Level 3 |
| Legal matters | ❌ | ✅ Always | Level 3 + urgent |
| Data sharing | ❌ | ✅ Always | Level 3 |

## Approval Request Template

```markdown
---
type: approval_request
request_id: APPROVAL_<timestamp>_<type>
action: <action_type>
created: <timestamp>
expires: <24h from now>
priority: <urgent|high|normal>
status: pending
risk_score: <number>
---

# Approval Required: <Action Type>

## Action Summary
**Action:** <what will happen>
**Target:** <recipient/system>
**Impact:** <expected outcome>

## Context
<Background information explaining why this action is needed>

## Proposed Action Details
<Specific details of what will be executed>

## Risk Assessment
- [ ] New contact (not in known contacts)
- [ ] Financial commitment: $<amount>
- [ ] Legal implications
- [ ] Contains sensitive information
- [ ] Reputation risk
- [ ] External communication

**Risk Score:** <X>/10

## Alternatives Considered
1. <Alternative 1>
2. <Alternative 2>

## To Approve
Move this file to: `/Approved/`

## To Reject
Move this file to: `/Rejected/` and add feedback below:

**Rejection Reason:**
<Leave blank if approving>

---
*Auto-generated by Personal AI Employee*
*Expires: <24h timestamp>*
```

## Expiration Handling

```yaml
expiration_policy:
  default: 24 hours
  urgent: 4 hours
  low_priority: 48 hours

  on_expire:
    action: escalate
    notify: true
    move_to: Expired/

  renewal:
    allowed: true
    max_renewals: 2
    renewal_period: 12 hours
```

## Error Handling

| Error | Recovery Action |
|-------|-----------------|
| Cannot write to Pending_Approval/ | Retry with backoff, alert human |
| Request ID conflict | Generate new ID with timestamp |
| Missing required fields | Prompt for missing information |
| Handbook rules unclear | Default to approval required |

## Security Considerations

### Never Do
- Auto-approve Level 3 actions
- Bypass approval for financial/legal matters
- Create backdated approval requests
- Delete approval history

### Always Do
- Include full context for decision
- Set appropriate expiration
- Log all approval requests
- Preserve audit trail
